## Простое приложение Python с GUI, Web‑интерфейсом и PostgreSQL

В проекте есть:
- **настольное GUI‑приложение** на `tkinter`;
- **web‑интерфейс** на `Flask`;
- работа с БД **PostgreSQL**;
- **тесты** на `pytest` для проверки web‑части.

### Файлы проекта
- **`app.py`** – десктопное GUI‑приложение (tkinter).
- **`web_app.py`** – веб‑приложение на Flask.
- **`templates/index.html`** – HTML‑шаблон для веб‑интерфейса.
- **`db.py`** – функции для подключения к PostgreSQL и работы с таблицей `contacts`.
- **`tests/test_web_app.py`** – pytest‑тесты web‑части (с заглушкой БД).
- **`requirements.txt`** – зависимости для Python.

### Настройка окружения
1. Установите Python 3.10+.
2. Установите PostgreSQL и создайте базу (например, `test_db`).
3. Создайте пользователя и выдать ему права на эту базу, либо используйте существующего `postgres`.
4. Установите зависимости (в PowerShell):

```bash
pip install -r requirements.txt
```

### Настройка подключения к БД
Подключение настраивается через переменные окружения (можно задать в PowerShell перед запуском):

```bash
$env:PG_HOST="localhost"
$env:PG_PORT="5432"
$env:PG_DB="test_db"
$env:PG_USER="postgres"
$env:PG_PASSWORD="ваш_пароль"
```

Если переменные не заданы, будут использованы значения по умолчанию:
- хост: `localhost`
- порт: `5432`
- база: `test_db`
- пользователь: `postgres`
- пароль: `postgres`

### Запуск GUI‑приложения

В каталоге проекта выполните:

```bash
python app.py
```

При первом запуске автоматически создаётся таблица `contacts` (если её ещё нет).

### Запуск web‑приложения

В том же каталоге выполните:

```bash
python web_app.py
```

По умолчанию приложение будет доступно в браузере по адресу:

```text
http://127.0.0.1:5000/
```

На странице можно:
- **просматривать** список контактов;
- **добавлять** новый контакт;
- **редактировать** существующие контакты;
- **удалять** контакты.

### Запуск тестов

Для запуска тестов (web‑слоя, без реальной PostgreSQL) выполните:

```bash
pytest
```
